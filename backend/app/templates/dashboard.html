      LangGraph Multi-Agent Architecture

      This platform runs an autonomous multi-agent workflow using LangGraph. Here is the sequence of parallel and serial node executions triggered when you analyze a stock ticker:

      Entry Node
      Financial Agent

      Fetches real-time price, history (yfinance) and fundamentals.

      Parallel Execution
      News & Sentiment Flow
      News Agent (NewsAPI/Tavily) → Sentiment Agent (FinBERT Tone Classification) → ML Predictor (XGBoost 30-Day prediction)
      Risk Profile Node
      Calculates annualized volatility, Nifty Beta, and Sharpe ratio.
      RAG Audit Node
      Retrieves context from uploaded annual reports via ChromaDB vector store.
      Join Node (End)
      Reporter Agent

      Aggregates all results and runs Gemini 2.5 Flash to compile markdown advisory report.

      Graph State Coordination Details:
      • State Initialization: The supervisor receives the ticker, seeds a blank `execution_logs` array, and saves an initial database run to obtain a unique `analysis_id`.
      • Data Dependencies: News, Risk, and RAG agents execute concurrently because they only depend on initial metadata returned by the Financial Agent.
      • NLP Sentiment & ML Sequence: The Sentiment Agent runs immediately after the News Agent finishes classifying headlines, and its output is then used by the XGBoost Predictor to run feature importance (SHAP).
      • Reporter Coalition: The Reporter Agent acts as the final sink, executing only after all parallel runs finish to compile the Markdown advisory report and save the consolidated state to SQLite.
